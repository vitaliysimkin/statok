# Prompt 01 — Збір вимог через /design-decisions

**Роль агента:** фінансовий продакт-аналітик. Поєднує доменну фінансову експертизу (інвестиції, портфельний облік, мультивалюта, податки/звітність) з продуктовим мисленням (`design:user-research`, `product-management:product-brainstorming`).

**Завдання:** через скіл `/design-decisions` згенерувати робочий файл `specs/statok-requirements.decisions.md` на **20–30 питань**, що покривають вимоги до продукту Statok та підтверджують технічні рішення стеку/деплою.

**Контекст продукту:** self-hosted, single-user. Фаза 1 — інвестиції/портфель (акції/ETF, крипта, облігації, депозити/готівка, мультивалюта). Фаза 2 — витрати + аналітика. Пріоритет — приватність даних.

**Вимоги до набору питань (групувати за темами):**
1. Скоуп і фазування (MVP, single/multi-user, платформа).
2. Інвестиції: класи активів, спосіб вводу (manual/CSV/API), мультивалюта й базова валюта, джерело котирувань, крипта, облігації (купон/погашення/YTM), дивіденди/реінвест, метрики дохідності (TWR/IRR/P&L), cost-basis (FIFO/avg), історія net worth.
3. Витрати (фаза 2): спосіб логування, категоризація, бюджети.
4. Аналітика/звітність: дашборди, експорт (PDF/Excel/податковий), алерти (Telegram).
5. Дані/інтеграції: формати імпорту, баланс автоматизація/ручний ввід, підтвердження приватності.
6. Техніка/деплой (з `research/deployment-blueprint.md`): template = tardis? Bun vs npm? Hetzner+Traefik+домен? зберігання грошей (integer minor units)? провайдер котирувань/FX? Telegram-нотифай?

**Формат:** кожен Q-блок — англомовні ключові слова (`Options`, `Recommendation`, `Answer`, `Comment`), контент укр. Combinable там, де можна кілька варіантів. Для безальтернативних — `[assumption]`. Для кожного — рекомендація в 1 речення.

**Резолв:** ≥3 питань → web UI (`server.mjs`), `TASKS_DIR` = `O:\projects\statok\specs`. Чекати маркер/«готово».

**Вихід:** після відповідей — дистиляція в `specs/requirements.md` (секція `## Decisions`), видалення `.decisions.md`. Далі → Prompt 02.
