# Statok — Вимоги (узгоджені рішення)

Self-hosted, single-user застосунок особистих фінансів. Фаза 1 — облік інвестицій/портфеля, фаза 2 — витрати + аналітика. Пріоритет — приватність: дані лише на власній інфраструктурі.

Рішення зафіксовані 2026-06-11 через design-decisions (30 питань, відповіді власника). Це джерело істини для ТЗ (`statok-tz.md`).

## Контекст власника

- Зараз у портфелі **лише ОВДП** (акцій/крипти поки нема, але плануються) → **облігаційний модуль — найвищий пріоритет** серед класів активів.
- Рахунки в **UAH, USD, EUR**; базова валюта звітності — **USD**.
- Філософія MVP: мінімум зараз, архітектура не повинна заважати розширенню (метрики, імпорт, алерти — додамо потім).

## Decisions

### Скоуп і платформа

- **MVP (фаза 1)**: тільки інвестиції/портфель — активи, транзакції, P&L, net worth. Без витрат і без API-синку брокерів.
- **Single-user + приватність**: один акаунт, без шарингу; зовнішні виклики лише до цінових/курсових API; Telegram-нотифікації несуть мінімум даних.
- **Платформа**: веб + PWA (встановлюється на телефон, базовий офлайн-кеш app shell); десктоп-первинний дизайн, адаптив.

### Інвестиційне ядро

- **Класи активів (фаза 1)**: акції/ETF, криптовалюта, облігації (ОВДП), депозити/готівка/валюта. Нерухомість/альтернативи — пізніше.
- **Рахунки — перший клас**: кожна транзакція й позиція належить рахунку (брокер/біржа/банк/гаманець); алокація по інституціях.
- **Ввід даних (v1)**: ручний ввід транзакцій + механізм **«поточного стану»** (opening balance/position: завести наявну позицію без повної історії угод, із опційною собівартістю). CSV-імпорт у v1 НЕ входить.
- **Комісії**: поле «комісія» на кожній угоді — включається в собівартість купівлі / зменшує виручку продажу. Окремого типу «періодичний збір» у v1 нема.
- **Корпоративні дії**: типи «спліт» і «зміна тікера» зі збереженням історії — реалізувати мінімально, **низький пріоритет** (власник поки без акцій).
- **Перекази між рахунками**: звʼязана пара transfer-out/transfer-in — внески на брокера не псують кешфлоу/дохідність. In-kind переказ активів — поза v1.
- **Базова валюта**: USD (конфіг `BASE_CURRENCY`); позиції живуть у своїх валютах, net worth і звіти конвертуються в базову.
- **Джерела цін**: Yahoo Finance (неофіційний API) для акцій/ETF/крипти (пари типу `BTC-USD`) + **ручне редагування ціни** як універсальний фолбек. CoinGecko/платні API — не зараз.
- **Облігації — повна деталізація**: номінал, купон, частота виплат; дата погашення з автопогашенням у кеш; розрахунок YTM/поточної дохідності.
- **Дивіденди/купони**: виплати з полями брутто / утриманий податок / нетто, історія доходу; реінвест (DRIP) — опційно.
- **Метрики дохідності (v1)**: проста P&L (реалізована/нереалізована, абсолютна і %). XIRR/TWR/бенчмарк — свідомо пізніше (модель транзакцій із переказами вже їх уможливлює).
- **Собівартість**: середньозважена (average cost). FIFO — можливий апгрейд.
- **Net worth історія**: денні снапшоти фоновою джобою (EOD ціни + курси) + кнопка ручного перерахунку. Некритично для першого запуску — можна останньою чергою фази 1.

### Фаза 2 — витрати (рамка, деталізація перед стартом фази)

- Ручний ввід (PWA з телефону), імпорт CSV-виписок банків, **авто-синк monobank** (офіційний особистий API), категорії пресет + кастомні. Авто-правила і бюджети — після обкатки.

### Аналітика, звіти, алерти

- **Дашборди v1**: (1) net worth у часі — головний графік; (2) грошові потоки (внески/виведення/купони/дивіденди по періодах). Алокація і performance-дашборди — пізніше. Операційні екрани (рахунки, позиції, транзакції) — базовий CRUD, не «аналітика».
- **Експорт**: CSV транзакцій/позицій (також страховка від lock-in).
- **Податки**: поза скоупом v1 — тільки realized P&L у базовій валюті. Денна історія курсів НБУ в БД (див. FX) лишає двері для річного звіту в UAH без міграції даних.
- **Алерти**: лише Telegram-канал як інфраструктура (деплой-нотифікації за патерном tardis). Продуктові алерти (цінові, дрейф) — поза v1.

### Дані, безпека, UX

- **Імпорт-архітектура (на майбутнє)**: генерик CSV-маппер (збережувані маппінги колонок) + пресети під конкретні джерела — закладається після MVP; конкретні брокери/банки ще не визначені.
- **Бекапи**: нічний `pg_dump` на VPS + шифрована (age/gpg) копія у віддалений сторедж (B2/S3 через rclone) + кнопка «завантажити бекап» в UI.
- **Автентифікація**: логін/пароль (bcrypt-сід адміна з `.env`) + JWT — патерн tardis. 2FA/passkey — можливо пізніше.
- **Локалізація**: i18n (vue-i18n), українська за замовчуванням + англійська.
- **Тема**: темна + світла, перемикач, авто за системою. Без onboarding-майстра і demo seed-даних.

### Стек і деплой (дзеркало tardis)

- **Стек**: Bun + Hono + Drizzle ORM + Postgres 16 (бек); Vue 3 + Vite + nginx (фронт); **Bun усюди** включно з фронтом (npm-неузгодженість tardis не повторюємо). Монорепо `backend/ + frontend/ + packages/shared + infra/ + scripts/`.
- **Гроші в БД**: гібрид — фіат-суми як **integer minor units + код валюти**; кількості активів, ціни, курси — **numeric** (дробові акції/сатоші/wei). Округлення half-up до мінорної одиниці на рівні звітів.
- **FX**: Frankfurter/ECB (крос-курси) + **НБУ API** (офіційний курс UAH); денна історія курсів зберігається в БД.
- **Деплой**: той самий Hetzner VPS за спільним Traefik (`web` network), домени `statok.simk.in.ua` + `api.statok.simk.in.ua`, GHCR + GitHub Actions, реліз по тегу `vX.Y.Z`, Telegram-нотифай. Секрети — лише на VPS (`/opt/statok/.env`). Dev-порти: Postgres `5434`, бек `3100`, Vite `5273`.

## Відкриті питання (не блокують ТЗ)

1. Точна UX-механіка «поточного стану» (opening positions): окремий тип транзакції чи майстер на рахунку — вирішити в ТЗ.
2. Перелік CSV-пресетів імпорту — власник вкаже джерела, коли дійде черга (фаза 1.5+).
3. Депозити: зі ставкою/строком чи просто баланс — у ТЗ закласти просту модель балансу + опційні поля ставки/строку.
